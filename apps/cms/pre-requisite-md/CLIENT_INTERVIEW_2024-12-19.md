## 🆕 Phase 6: 공문 관리 (신규 추가)

### 6.1 공문/홍보 프로세스

**현재 상황:**
- 공문으로 프로그램 홍보 진행
- 인스타 등에서도 공문처럼 **프로그램별 개별 홍보** 진행
- 한 학교에 공문 여러 개가 갈 수 있음 (시작일이 달라서)
- 프로그램별 모집 시기가 다름
- 현재는 프로그램별로 **개별 송부**

**요구사항:**
- 하나의 공문에 프로그램 리스트가 이어져도 됨 ✅
- 공문 **템플릿화** 가능 여부 확인 필요
- **PDF / Word 파일로 다운로드** 가능 여부 확인 필요
- 공문상에 구체 프로그램이 안 들어나도 **학교 정책적으로 공문 처리 가능한지** 확인 필요

**UI STRUCTURE REQUIREMENTS:**
- [ ] 공문 목록 페이지
- [ ] 공문 생성/수정 폼
  - 프로그램 선택 (다중 선택 가능)
  - 템플릿 선택
  - 프로그램 리스트 자동 구성
- [ ] 공문 미리보기
- [ ] 공문 다운로드 (PDF/Word)
- [ ] 공문 템플릿 관리

**FORBIDDEN:**
- 공문 내용 자동 생성 (수동 입력 필수)
- 학교 정책 검증 로직 (외부 확인 필요)

## 🆕 Phase 7: 신청 경로 관리 (신규 추가)

### 7.1 신청 경로 설정

**현재 상황:**
- 구글 forms 링크 발송
- 신청 자체는 **업무 자동화 프로그램에서**
- 학교 프로그램과 개인 프로그램이 다른데 **양방향 신청이 불가능**

**요구사항:**
- 프로그램별 신청 경로 설정
  - 구글폼 링크
  - 자동화 프로그램 내 신청
- 학교 프로그램 vs 개인 프로그램 구분
- 신청 경로별 안내 문구 설정

**UI STRUCTURE REQUIREMENTS:**
- [ ] 프로그램별 신청 경로 설정 UI
- [ ] 구글폼 링크 관리
- [ ] 신청 경로별 안내 문구 설정

**FORBIDDEN:**
- 양방향 신청 동시 활성화
- 신청 경로 자동 변경

## 🆕 Phase 11: 정산 관리 고도화 (신규 추가)

### 11.1 강사비 정산 플로우

**현재 상황:**
- 기존 정산이 어떻게 진행되는지(플로우) 확인 필요
- 오프라인 방식 → 온라인으로 전환
- 강의 진행 후 월단위 강사료 입금 (정해진 날짜)
- PM 1명이 프로그램 1개 + 다수 강사 관리
- 프로그램별 강사 수/강사별 강사료 상이

**지급 프로세스 요소:**
- 강사에게 지급조서 (월별)
- 내용/금액/계좌번호/주민등록번호/이름/전화번호/개인정보 동의 확인
- e-count 전자결제 첨부 / 이체신청
- 지급조서 문서 다운로드 가능하게
- 강사 입력 정보 기반
- 강의료 산출 자동화

**1419 사업 특수성:**
- 교통비/숙박비 지급
  - 숙박비: 타지역 이동 시 실비
  - 교통비: 거주지 기준 60km 초과 시 거리 기준
  - 룰 문서 존재 여부 → 텍스트로 정리해 받을 수 있음
- 정산 요청 주체: **JAKOREA에서 요청** (이 방식)

**자동화 플랫폼 운영 관점 아이디어:**
- 월별 관리가 담당자 입장에서 깔끔
- 총 강사 수/개별 정산액/필요 자료 요청 자동화 가능성
- 매번 승인 받기 어려움
- 최초 받은 정보로 지급정보 재사용 (권한자만 확인)
- 양방향 승인 시 결의 (담당자 확인 필요)

**요구사항:**
- [ ] 월별 정산 관리 화면
- [ ] 강사별 정산액 자동 산출
- [ ] 지급조서 생성/다운로드 (PDF)
- [ ] e-count 전자결제 연동 준비
- [ ] 교통비/숙박비 자동 계산 (1419 사업)
  - 거리 기준 계산 (60km 초과 시)
  - 타지역 이동 시 숙박비 실비
- [ ] 정산 승인 워크플로우
- [ ] 개인정보 동의 확인
- [ ] 지급정보 재사용 기능

**UI STRUCTURE REQUIREMENTS:**
- [ ] 월별 정산 목록 페이지
- [ ] 강사별 정산 상세 화면
- [ ] 정산 산출 로직 설정
  - 기본 강사료
  - 교통비 계산 규칙
  - 숙박비 계산 규칙
- [ ] 지급조서 생성 화면
  - 강사 정보 자동 입력
  - 개인정보 동의 확인
  - 금액 자동 계산
- [ ] 정산 승인 워크플로우
  - 담당자 확인
  - 최종 승인
- [ ] 지급조서 다운로드 (PDF)
- [ ] e-count 전자결제 연동 준비 (향후)

**FORBIDDEN:**
- 정산 자동 승인
- 개인정보 무단 노출
- 지급정보 무단 변경

## 🆕 Phase 12: 어드민 커스터마이징 (신규 추가)

### 12.1 콘텐츠 관리

**요구사항:**
- 어드민에서 이미지 업로드 등 콘텐츠 관리
- 메인 팝업 관리 (최대 3개 정도)
  - 사이즈 3가지로 정해 모듈 구조
- JAKOREA가 어떤 일을 하는지
  - 그래픽 이미지를 **사진으로 교체**하면 전달력 높아질 것
  - 어드민에서 커스터마이징 가능하면 좋음
- 임팩트 스토리
  - 사업 임팩트 존재
  - 어드민에서 이미지 업로드
  - 사업 결과가 기사성 컨텐츠처럼 느껴짐 → 컨텐츠 속성 변경 필요
    - 더 간결한 형태로 표현 필요

**UI STRUCTURE REQUIREMENTS:**
- [ ] 메인 팝업 관리 화면
  - 팝업 등록/수정/삭제
  - 사이즈 3가지 모듈 구조
  - 최대 3개 제한
- [ ] 콘텐츠 관리 화면
  - 이미지 업로드
  - 텍스트 편집
  - 사진 교체 기능
- [ ] 임팩트 스토리 관리 화면
  - 이미지 업로드
  - 간결한 형태로 표현
  - 사업 결과 표시

**FORBIDDEN:**
- 무제한 팝업 등록
- 이미지 자동 리사이징 (사이즈 제한 필요)

### 12.2 문의 관리

**요구사항:**
- 기업 후원문의 메일에 **알림 필요**
- 문의 내용이 이메일로 어드민에서 확인되면 좋겠다
- 문의 필드가 카테고리마다 나뉨
- 국장님 + 담당자 미팅 진행
- 문의 → 전화/회신
- 추적관리 니즈는 크지 않음 (있으면 좋긴 함)

**UI STRUCTURE REQUIREMENTS:**
- [ ] 문의 목록 페이지
  - 카테고리별 필터
  - 상태별 필터 (미처리/처리중/완료)
- [ ] 문의 상세 화면
  - 문의 내용 확인
  - 담당자 할당
  - 처리 상태 변경
  - 전화/회신 기록
- [ ] 문의 알림 기능
  - 이메일 알림
  - 어드민 내 알림
- [ ] 문의 추적 (선택사항)
  - 처리 이력
  - 담당자 변경 이력

**FORBIDDEN:**
- 문의 자동 처리
- 개인정보 무단 노출

---

### 12.3 다운로드 관리

**요구사항:**
- 다운로드 링크가 너무 작음 (링크) → 개선 필요
- 다운로드 페이지 중요
- 다운로드 카운트 (월별)로 실적 확인 필요
- 인재채용/뉴스레터/언론보도 비중 큼 (공지 비중 작음)
- 실적 중 하나가 컨텐츠 다운로드 수가 중요

**UI STRUCTURE REQUIREMENTS:**
- [ ] 다운로드 목록 페이지
  - 카테고리별 분류 (인재채용/뉴스레터/언론보도/공지 등)
  - 다운로드 링크 크기 개선
- [ ] 다운로드 통계 화면
  - 월별 다운로드 카운트
  - 카테고리별 통계
  - 파일별 통계
- [ ] 다운로드 파일 관리
  - 파일 업로드
  - 파일 정보 수정
  - 파일 삭제

**FORBIDDEN:**
- 다운로드 링크 자동 생성
- 통계 데이터 조작
