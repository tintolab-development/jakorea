**역할**: 기획자 (Planner)  
**주제**: 일정이 등록된 날짜 클릭 시 동작 결정

---

## 📋 현재 구현 상태

### 현재 동작
1. **빈 날짜 셀 클릭** → 일정 등록 모달 열림 (해당 날짜가 기본값)
2. **일정 아이템 클릭** → 일정 상세 drawer 열림
3. **일정이 있는 날짜 셀의 빈 영역 클릭** → 일정 등록 모달 열림

## 🤔 고려사항

### 1. 사용자 의도 분석

#### 시나리오 A: 일정이 있는 날짜의 빈 영역 클릭
- **의도 1**: "이 날짜에 또 다른 일정을 추가하고 싶다"
- **의도 2**: "이 날짜의 일정들을 보고 싶다" (실수로 빈 영역 클릭)

#### 시나리오 B: 일정이 있는 날짜의 일정 아이템 클릭
- **의도**: "이 특정 일정의 상세 정보를 보고 싶다"

### 2. 비즈니스 요구사항

- **같은 날 여러 일정 등록 가능 여부**: ✅ 가능 (같은 날 여러 프로그램/강의 가능)
- **일정 조회 빈도**: 높음 (일정 확인이 주요 사용 목적)
- **일정 등록 빈도**: 중간 (관리자가 주기적으로 등록)

## 💡 권장 UX 설계

### **옵션 1: 일정이 있는 날짜 → 상세 drawer만 열기 (권장)**

#### 동작
- 일정이 있는 날짜 셀 클릭 (빈 영역 포함) → **일정 상세 drawer 열림**
  - 여러 일정이 있는 경우: 첫 번째 일정 또는 일정 목록 표시
- 일정 등록은 **명시적 액션**으로만 가능:
  - "일정 등록" 버튼 클릭
  - 빈 날짜 셀 클릭

#### 장점
- ✅ **의도 명확성**: 날짜 클릭 = 조회 의도로 통일
- ✅ **실수 방지**: 의도치 않은 일정 등록 모달 방지
- ✅ **일관성**: 일정이 있는 날짜는 항상 조회 동작
- ✅ **UX 패턴**: 대부분의 캘린더 앱과 일치 (Google Calendar, Outlook 등)

#### 단점
- ❌ 같은 날 추가 일정 등록 시 "일정 등록" 버튼 클릭 필요

---

### **옵션 2: 일정이 있는 날짜 → 빈 영역 클릭 시 등록 모달 (현재 구현)**

#### 동작
- 일정 아이템 클릭 → 일정 상세 drawer
- 일정이 있는 날짜의 빈 영역 클릭 → 일정 등록 모달

#### 장점
- ✅ **빠른 추가**: 같은 날 일정 추가가 빠름
- ✅ **유연성**: 조회와 등록 모두 가능

#### 단점
- ❌ **의도 불명확**: 빈 영역 클릭 시 의도 파악 어려움
- ❌ **실수 가능성**: 조회하려다 등록 모달이 열릴 수 있음
- ❌ **UI 복잡성**: 클릭 영역에 따라 다른 동작

---

### **옵션 3: 하이브리드 (추천하지 않음)**

#### 동작
- 일정이 있는 날짜 클릭 → 일정 목록 drawer
- 일정 목록 drawer에 "일정 추가" 버튼 제공

#### 장점
- ✅ 조회와 등록 모두 가능
- ✅ 의도가 명확함

#### 단점
- ❌ 구현 복잡도 증가
- ❌ 추가 클릭 필요

## 🎯 최종 권장안: **옵션 1**

### 이유

1. **사용자 경험 일관성**
   - 대부분의 캘린더 앱은 날짜 클릭 = 조회
   - 사용자 기대와 일치

2. **의도 명확성**
   - 날짜 클릭 = "이 날짜의 일정을 보고 싶다"
   - 일정 등록 = 명시적 액션 (버튼 클릭)

3. **실수 방지**
   - 조회하려다 등록 모달이 열리는 실수 방지

4. **일정 조회가 주요 사용 목적**
   - 일정 확인이 등록보다 빈번한 작업

### 구현 변경사항

```typescript
// 일정이 있는 날짜 셀 클릭 시
onClick={e => {
  const target = e.target as HTMLElement
  if (!target.closest('.schedule-item')) {
    // 일정이 있는 경우: 첫 번째 일정의 상세 drawer 열기
    if (listData.length > 0) {
      onDateSelect(value, listData[0])
    } else {
      // 일정이 없는 경우: 일정 등록 모달 열기
      onDateSelect(value, undefined)
    }
  }
}}
```
