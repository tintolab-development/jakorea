## 🆕 Phase 5: 사용자 화면 기반 UI 개선 (신규 추가)

### 5.1 프로그램 상세 화면 개선 🆕

**참고 화면: U-01-02 프로그램 상세**

**ROLE:**

- CMS 관리자 화면 개발

**SCREEN PURPOSE:**

- 프로그램의 목적과 내용을 정확히 이해
- 프로그램이 승인제 기반임을 자연스럽게 인지
- 신청 가능 여부를 추측하지 않고 명확히 인식

**UI STRUCTURE REQUIREMENTS:**

- [ ] 프로그램 핵심 정보 영역 (상단)
  - 프로그램명 표시
  - 카테고리 표시
  - 프로그램 목적/설명 표시
- [ ] 대상 및 참여 방식 안내 영역
  - 대상 사용자 명확히 표시
  - 강사/봉사자 참여 필요 여부 설명
- [ ] 일정 요약 영역 (조건부)
  - 확정된 일정만 요약 형태로 표시
  - 미확정 일정 노출 금지
- [ ] 신청 안내 및 상태 영역 (핵심)
  - 승인제 고정 문구 포함
  - 신청 가능 여부 표시 (서버 응답 기반)
- [ ] 보조 안내 영역
  - 신청 전 사용자 불안을 줄이기 위한 문구

**FORBIDDEN:**

- 프로그램 상태 추론
- 신청 가능 여부를 프론트엔드에서 판단
- 승인제 누락 또는 축소 설명
- 일정 미확정 정보 노출
- 신청 버튼을 조건 없이 노출
- 관리자/운영 정보 노출

### 5.3 마이페이지 메인 화면 개선 🆕

**참고 화면: U-03-01 마이페이지 메인**

**ROLE:**

- CMS 관리자 화면 개발

**SCREEN PURPOSE:**

- 사용자가 현재 자신의 전체 상태를 즉시 이해
- 지금 해야 할 일이 있는지/없는지를 헷갈리지 않음
- 승인 대기·반려·진행·종료 등 모든 상태를 문장으로 설명

**UI STRUCTURE REQUIREMENTS:**

- [ ] 페이지 헤더 영역
  - 페이지 제목: "마이페이지"
  - 보조 설명: "신청 현황과 일정, 이력을 한눈에 확인할 수 있습니다."
- [ ] 상태 요약 영역 (최상단, 가장 강조)
  - primaryStatus 기반으로 단 하나의 상태만 표시
  - 상태는 반드시 문장으로 설명
  - APPLY_01(승인 대기)일 때 필수 문구 포함
  - reason_public이 있으면 반드시 그대로 표시
- [ ] To-do 영역
  - To-do는 최대 2개만 노출, 우선순위 순으로 표시
  - 각 To-do 항목: label 표시 + CTA 1개 [바로 처리하기]
  - todos가 비어 있을 경우: "현재 처리하실 작업이 없습니다."
- [ ] 내 일정 요약 영역 (조건부)
  - upcomingSchedules가 존재할 경우에만 노출
  - 승인 완료된 일정만 표시
- [ ] 내 이력 요약 영역 (조건부)
  - historySummary가 존재할 경우에만 노출
  - 최대 3개까지만 요약 노출
  - CTA 단일: [전체 이력 보기]
- [ ] 보조 안내 영역
  - "상태 변경 및 결과 안내는 서비스 알림과 함께 마이페이지에서 확인할 수 있습니다."

**FORBIDDEN:**

- 상태(Enum) 추론 또는 생성
- 관리자 로직/데이터 노출 (reason_internal, 로그 등)
- To-do 3개 이상 노출
- 복수 주요 CTA 동시 노출
- "문의하세요/연락하세요" 문구

- **단일 상태만 표시**: 복수 상태 동시 노출 금지
- **Enum 기반 조건 제어**: 하드코딩 금지
- **reason_public 그대로 표시**: 의미 변경·요약·재작성 금지

### CTA 원칙

- **최대 1개만 노출**: 복수 주요 CTA 동시 노출 금지
- **명확한 행동 안내**: 사용자가 "지금 뭘 해야 하지?"라고 고민하면 실패
- **targetUrl 기반 네비게이션**: 프론트엔드 임의 판단 금지

### 안내 문구 원칙

- **고정 문구 준수**: 의미 변경·요약 금지
- **순서 변경 불가**: 문장 순서는 반드시 유지
- **안정적인 어조**: 사용자 불안 유발 표현 금지

### FORBIDDEN 원칙

- **상태(Enum) 추론 또는 생성**: 서버 응답만 사용
- **관리자/운영 정보 노출**: reason_internal, 로그 등 노출 금지
- **문의 유도 문구**: "문의하세요", "담당자 연락" 등 금지
- **복수 주요 CTA 동시 노출**: 항상 단일 CTA
