### 4.2 정산 관리 고도화 🆕 (인터뷰 기반)

**참고**: `CLIENT_INTERVIEW_2024-12-19.md` Phase 11

**ROLE:**

- CMS 관리자 화면 개발

**SCREEN PURPOSE:**

- 월별 정산 관리 (담당자 입장에서 깔끔한 관리)
- 강사별 정산액 자동 산출
- 지급조서 생성/다운로드 (PDF)
- e-count 전자결제 연동 준비
- 교통비/숙박비 자동 계산 (1419 사업 특수성)
- 정산 승인 워크플로우
- 개인정보 동의 확인
- 지급정보 재사용 기능

**현재 상황:**

- 기존 정산이 어떻게 진행되는지(플로우) 확인 필요
- 오프라인 방식 → 온라인으로 전환
- 강의 진행 후 월단위 강사료 입금 (정해진 날짜)
- PM 1명이 프로그램 1개 + 다수 강사 관리
- 프로그램별 강사 수/강사별 강사료 상이

**지급 프로세스 요소:**

- 강사에게 지급조서 (월별)
- 내용/금액/계좌번호/주민등록번호/이름/전화번호/개인정보 동의 확인
- e-count 전자결제 첨부 / 이체신청
- 지급조서 문서 다운로드 가능하게
- 강사 입력 정보 기반
- 강의료 산출 자동화

**1419 사업 특수성:**

- 교통비/숙박비 지급
  - 숙박비: 타지역 이동 시 실비
  - 교통비: 거주지 기준 60km 초과 시 거리 기준
  - 룰 문서 존재 여부 → 텍스트로 정리해 받을 수 있음
- 정산 요청 주체: **JAKOREA에서 요청** (이 방식)

**자동화 플랫폼 운영 관점 아이디어:**

- 월별 관리가 담당자 입장에서 깔끔
- 총 강사 수/개별 정산액/필요 자료 요청 자동화 가능성
- 매번 승인 받기 어려움
- 최초 받은 정보로 지급정보 재사용 (권한자만 확인)
- 양방향 승인 시 결의 (담당자 확인 필요)

**UI STRUCTURE REQUIREMENTS:**

- [ ] 월별 정산 목록 페이지
  - 월별 필터
  - 프로그램별 필터
  - 총 강사 수 표시
  - 개별 정산액 요약
- [ ] 강사별 정산 상세 화면
  - 강사 정보 표시
  - 정산 산출 내역 (기본 강사료, 교통비, 숙박비)
  - 개인정보 동의 확인 UI
  - 지급정보 표시 (계좌번호, 주민등록번호 등)
- [ ] 정산 산출 로직 설정
  - 기본 강사료 설정
  - 교통비 계산 규칙 (60km 초과 시)
  - 숙박비 계산 규칙 (타지역 이동 시 실비)
  - 1419 사업 특수성 반영
- [ ] 지급조서 생성 화면
  - 강사 정보 자동 입력
  - 개인정보 동의 확인
  - 금액 자동 계산
  - 지급조서 미리보기
- [ ] 정산 승인 워크플로우
  - 담당자 확인 단계
  - 최종 승인 단계
  - 양방향 승인 시 결의 UI
- [ ] 지급조서 다운로드 (PDF)
  - 지급조서 템플릿
  - PDF 생성 및 다운로드

  - 연동 인터페이스 설계
  - 데이터 포맷 정의

- [ ] 문의 알림 기능
  - 이메일 알림
  - 어드민 내 알림
- [ ] 문의 추적 (선택사항)
  - 처리 이력
  - 담당자 변경 이력

**FORBIDDEN:**

- 문의 자동 처리
- 개인정보 무단 노출

**확인 필요 사항:**

- 카테고리별 필드 구조 확인
- 알림 방식 확인 (이메일/어드민)

### 11.3 다운로드 관리 🆕

**ROLE:**

- CMS 관리자 화면 개발

**SCREEN PURPOSE:**

- 다운로드 파일 관리
- 다운로드 통계 (월별, 카테고리별, 파일별)
- 다운로드 링크 크기 개선

**요구사항:**

- 다운로드 링크가 너무 작음 (링크) → 개선 필요
- 다운로드 페이지 중요
- 다운로드 카운트 (월별)로 실적 확인 필요
- 인재채용/뉴스레터/언론보도 비중 큼 (공지 비중 작음)
- 실적 중 하나가 컨텐츠 다운로드 수가 중요

**UI STRUCTURE REQUIREMENTS:**

- [ ] 다운로드 목록 페이지
  - 카테고리별 분류 (인재채용/뉴스레터/언론보도/공지 등)
  - 다운로드 링크 크기 개선
  - 파일 업로드
  - 파일 정보 수정
  - 파일 삭제
- [ ] 다운로드 통계 화면
  - 월별 다운로드 카운트
  - 카테고리별 통계
  - 파일별 통계
  - 차트/그래프 표시

**FORBIDDEN:**

- 다운로드 링크 자동 생성
- 통계 데이터 조작

## 🎨 공통 UI 원칙 (사용자 화면 프롬프트 기반)

### 상태 표시 원칙

- **상태는 반드시 문장으로 설명**: 아이콘/색상보다 텍스트 설명 우선
- **단일 상태만 표시**: 복수 상태 동시 노출 금지
- **Enum 기반 조건 제어**: 하드코딩 금지
- **reason_public 그대로 표시**: 의미 변경·요약·재작성 금지

### CTA 원칙

- **최대 1개만 노출**: 복수 주요 CTA 동시 노출 금지
- **명확한 행동 안내**: 사용자가 "지금 뭘 해야 하지?"라고 고민하면 실패
- **targetUrl 기반 네비게이션**: 프론트엔드 임의 판단 금지

### 안내 문구 원칙

- **고정 문구 준수**: 의미 변경·요약 금지
- **순서 변경 불가**: 문장 순서는 반드시 유지
- **안정적인 어조**: 사용자 불안 유발 표현 금지

### FORBIDDEN 원칙

- **상태(Enum) 추론 또는 생성**: 서버 응답만 사용
- **관리자/운영 정보 노출**: reason_internal, 로그 등 노출 금지
- **문의 유도 문구**: "문의하세요", "담당자 연락" 등 금지
- **복수 주요 CTA 동시 노출**: 항상 단일 CTA

### 즉시 확인 필요

1. **공문 템플릿화**
   - 템플릿 구조 정의 필요
   - PDF/Word 다운로드 포맷 확인

   - 담당자 실무 확인 필요 → 플로우 전달받아야 함
   - e-count 전자결제 연동 방식 확인

3. **1419 사업 특수성**
   - 교통비/숙박비 계산 룰 문서 전달 필요
   - 거리 계산 기준 확인

4. **수료증 배경 이미지**
   - 변경 가능 여부 확인
   - 이미지 포맷/크기 제한 확인

### 추가 확인 필요

1. **예산 관리**
   - 프로그램별 예산 존재 여부 확인
   - 예산 집계 방식 확인

2. **실적 집계**
   - 월별 집계 기준 확인
   - 농어촌 등 특수 기준 확인

3. **문의 관리**
   - 카테고리별 필드 구조 확인
   - 알림 방식 확인 (이메일/어드민)
