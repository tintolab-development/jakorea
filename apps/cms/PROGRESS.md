# CMS 프로젝트 진행 상황

이 문서는 CMS 프로젝트의 진행 상황을 롤별로 기록합니다.

## 📋 진행 기록 형식

각 항목은 다음 형식으로 기록합니다:

- **프롬프트/요청**: 어떤 작업을 요청했는지
- **결과**: 어떤 결과가 도출되었는지
- **롤별 역할**:
  - 🎨 **디자이너**: 디자인 관련 작업
  - 📋 **기획자**: 기획 및 요구사항 정의
  - 👨‍💼 **PM**: 일정 관리 및 의사결정
  - 👨‍💻 **개발자**: 개발 및 구현

## Phase 0: 프로젝트 초기 설정 (완료)

### 2024년 - 프로젝트 생성 및 기본 설정

**프롬프트/요청**:

- 모노레포에 vite/react 환경으로 cms 프로젝트 추가
- Ant Design 설치 및 기본 설정
- 프로젝트 가이드 문서 작성
- 필수 라이브러리 설치
- Mock 데이터 구조 생성 (30-50개 이상의 다양한 상태 데이터)

**결과**:

- ✅ CMS 프로젝트 생성 완료 (`apps/cms`)
- ✅ Ant Design 5.28.0 설치 및 ConfigProvider 설정
- ✅ PROJECT_GUIDE.md 작성 (FSD 구조, 컴포넌트 관심사 분리, Custom Hooks 가이드 포함)
- ✅ 필수 라이브러리 설치 완료 (react-router-dom, zustand, react-hook-form, zod, @tanstack/react-table 등)
- ✅ Mock 데이터 구조 생성 완료 (총 310개 데이터)
  - 스폰서: 30개
  - 학교: 30개
  - 강사: 50개
  - 프로그램: 40개
  - 신청: 50개
  - 일정: 40개
  - 매칭: 35개
  - 정산: 35개
- ✅ 반응형 디자인 가이드: 최소 1920x1080 데스크탑만 지원
- ✅ Locale 관리 제거: 한국어만 지원

**롤별 역할**:

- 👨‍💻 **개발자**: 프로젝트 구조 설계, 라이브러리 설치, Mock 데이터 생성 로직 구현
- 📋 **기획자**: Mock 데이터 요구사항 정의 (30-50개, 다양한 상태)
- 🎨 **디자이너**: 반응형 디자인 범위 정의 (1920x1080 데스크탑만)
- 👨‍💼 **PM**: 프로젝트 초기 설정 승인 및 진행 방향 결정

**참고사항**:

- Mock 데이터는 상황별로 수정 가능하도록 설계됨
- 각 Mock 데이터 파일은 독립적으로 수정 가능
- Map 기반 조회 구조로 성능 최적화

**다음 단계 브리핑** (짧은 버전):

- **작업**: 강사 목록 테이블 디자인, 강사 등록/수정 폼 디자인, 강사 상세 정보 카드 디자인
- **제약**: Ant Design Table, Form 컴포넌트 활용, 1920x1080 고정 레이아웃
- **산출물**: 강사 목록/상세/폼 디자인 시안
- **예상**: 2-3일

#### 📋 기획자 브리핑

- **작업**: 강사 관리 기능 명세서 작성 (목록 조회, 필터, 정렬, 페이지네이션, 상세 조회, 등록/수정)
- **참고**: MVP_ROADMAP.md Phase 1.2, Mock 데이터 구조 (50개 강사 데이터)
- **산출물**: 강사 관리 기능 명세서, 사용자 시나리오
- **예상**: 2-3일

#### 👨‍💼 PM 브리핑

- **작업**: Phase 1.2 일정 확정, 디자인/기획 산출물 승인
- **일정**: 전체 1주 (디자인 2-3일, 기획 2-3일, 개발 3-5일)
- **체크포인트**: 강사 관리 UI 디자인 승인, 기능 명세서 승인

#### 👨‍💻 개발자 브리핑

- **작업**: 강사 목록 페이지 (@tanstack/react-table), 강사 상세 페이지, 강사 등록/수정 폼 (react-hook-form + zod), Zustand 스토어 구현
- **준비**: 디자인 시안, 기능 명세서 확정 대기
- **예상**: 3-5일 (디자인/기획 완료 후)
