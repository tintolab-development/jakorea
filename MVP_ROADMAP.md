# JaKorea LMS MVP 개발 로드맵

## 📋 MVP 범위 (Mock 데이터 기반)

### ✅ 포함 기능

- 프로그램 관리 (Program)
- 신청 관리 (Application)
- 강사 매칭 (Matching)
- 캘린더 기반 일정 관리 (Schedule/Calendar)
- 강사 DB (Instructor)
- 정산 관리 (Settlement)
- 관리자 대시보드

### ❌ 제외 기능 (1차 MVP)

- 백엔드 API 연동 (Mock 데이터 사용)
- 외부 공개 홈페이지
- 고급 통계/대시보드 (기본 집계만)
- 다국어 지원
- 모바일 앱 (반응형 웹은 포함)

### Phase 6: 강사 정산 제출 기능 (1주) - 신규 추가

#### 6.1 강사 정산 제출 폼 (구글폼 스타일)

**기획 요구사항:**

- 강사가 강의 완료 후 정산 정보 제출
- 프로그램/강의 선택
- 비용 항목 입력 (강사비, 교통비, 숙박비, 유류비 등)
- 증빙 파일 업로드 (유류비 증빙 필수)
- 총액 자동 계산
- 제출 후 상태 관리 (pending → calculated → approved)

**디자인 요구사항:**

- 구글폼 스타일 UI (섹션 카드 레이아웃)
- M3 컴포넌트 활용 (TextField, Select, Button, Card)
- 파일 업로드 UI (드래그앤드롭 느낌)
- 총액 실시간 계산 표시
- 제출 완료 피드백

**개발 작업:**

- [x] 강사 정산 제출 폼 페이지
- [x] 프로그램/강의 선택 기능 (Matching 기반)
- [x] 비용 항목 입력 (강사비, 교통비, 숙박비, 유류비)
- [x] 파일 업로드 기능 (증빙 자료)
- [x] 총액 자동 계산 로직
- [x] 제출 데이터 처리 (Settlement 생성, 상태: 'pending')
- [x] 라우팅 설정 (`/settlements/submit`)

**산출물:**

- 강사 정산 제출 폼 페이지
- 파일 업로드 컴포넌트
- 제출 데이터 처리 로직

**참고:**

- Google Forms API는 직접 연동 불가능하므로 구글폼 스타일의 폼을 직접 구현
- 증빙 파일은 로컬 파일 객체로만 처리 (Mock 단계에서는 실제 업로드 없음)
- 제출된 데이터는 Settlement 엔티티로 변환되어 관리자 승인 대기 상태로 설정
